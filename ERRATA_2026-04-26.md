# Erratum — License audit, 2026-04-26

A license audit on 2026-04-26 found six material errors and one
provenance error in the SBOM data published in `releases/RC1-UAT1.1`
through `releases/RC1-UAT1.16` prior to today. The
`rAIdio.bot-RC1-UAT1.16.cdx.json` file in the `RC1-UAT1.16/`
directory has been **regenerated today with the corrected data**.
Earlier RCs (1.1 through 1.15) carried the same incorrect data and
have not been republished — see "What this erratum applies to"
below.

### Bundled-component caveat (worth knowing)

The `opencv-python-headless` wheel ships its own LGPL-2.1 ffmpeg
internally for `cv2.VideoCapture` support. That bundled ffmpeg is
**separate** from the GPL-3.0-or-later ffmpeg listed under
"GPL-3.0" in `THIRD_PARTY_NOTICES.txt` (which we ship as our
top-level audio tool). The build therefore contains two ffmpeg
binaries under two different effective licences. This is now
documented in the THIRD_PARTY_NOTICES.txt entry for
`opencv-python-headless`.

### Upstream-side inconsistencies (NOT errata for us)

For the record, these are inconsistencies inside the upstream
projects' own metadata. We follow the canonical github LICENSE in
each case, and our SBOM is internally consistent with that choice.
Listed here so consumers can see we noted them:

- **ACE-Step v1.5** — `github.com/ace-step/ACE-Step` LICENSE is
  Apache-2.0; the HF model card `ACE-Step/Ace-Step1.5` YAML
  frontmatter declares `license: mit`. We follow the github
  LICENSE.
- **Whisper** — `github.com/openai/whisper` LICENSE is MIT; the HF
  card `openai/whisper-small` YAML declares `license: apache-2.0`.
  We follow the github LICENSE.

## What this erratum applies to

The same incorrect data was published in the SBOMs for RC1-UAT1.1
through RC1-UAT1.16 prior to today (2026-04-26). Only the
`RC1-UAT1.16/` CycloneDX has been regenerated with corrected data
in this same commit; the timestamp inside that file reflects the
regeneration (not the original release date).

Older RCs (1.1 .. 1.15) have NOT been retroactively edited — those
artefacts continue to document what was claimed at the time of
each release, with this erratum as the authoritative correction
record. Consumers of those older SBOMs should treat this file as
the supplement.

## Audit method

Each material correction above was verified against the canonical
upstream LICENSE file directly (raw GitHub URL or the github API
`license.spdx_id` field) by two independent reviewers on
2026-04-26 (claims cross-checked between two LLM agents reading
the upstream sources independently). The procurement-decision
items (SeedVC-ComfyUI, ComfyUI-Demucs-AudioSeparator) were
verified by inspecting the upstream repo's file tree and
confirming the absence of any LICENSE file.

## Methodology change

The audit incident is documented internally as "incident 35" in
the source repo. As a follow-up, the rule for license reporting
has been formalised:

> When sources conflict, the canonical LICENSE file in the project's
> source repository wins. Distribution-platform metadata (HF YAML,
> PyPI classifiers, npm `package.json` license field, mirror-fork
> LICENSE files) is supplementary; the canonical file wins unless
> the upstream explicitly carves out different terms in prose.

This rule is applied retroactively in this erratum and prospectively
on every future SBOM regeneration.
